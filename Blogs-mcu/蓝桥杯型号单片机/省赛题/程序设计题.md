# 12届省赛
### 心得
1. Key_Proc(),Seg_Proc(),Led_Proc()函数之间靠全局变量相互联系，各种功能的实现实际上是在不同模块里改变全局变量的值。
2. 按键处理函数同样使用switch语句实现不同按键的不同功能。
3. 数码管显示函数中数据要先获取（例如：Temperature = rd_temperature();），然后再赋值给Seg_Buf[]数组，否则可能会造成频闪。
4. 整数小数转换尽量使用强制转换，不使用中间变量。
5. 参数设置类的题目需要两个变量，即一个显示的变量，一个实际控制的变量，只有在满足某一条件(如退出设置界面或按下保存键)实际控制变量才等于显示变量。
6. unsigned char 0-255
# 13届省赛
### 心得
1. 续12届模板第一条：对Seg_Buf[],ucLed[]等数组的赋值一般是靠变量完成的，而按键等一系列操作改变的是变量值，通过变量构建起操作与现象之间的桥梁。
2. 要通过尽可能少的变量实现功能，变量名要见名知意，在使用变量赋值时要清楚变量之间的逻辑关系。
3. 可以用三目运算符，逻辑表达式(Temperature > Temperature_Disp)代替if-else语句，用异或表达式完成0/1转换。
4. 1ms的定时器扫描大致可理解成实时扫描，因此一些模块函数可以不在Led_Proc()函数中和各种判断混在一起，可以写在中断函数里，而Led_Proc()函数里只用通过判断条件改变变量值即可。
